## 1. Purpose of This Document

This document defines the **hard scope boundaries** for **VP-Flow v1.1-B** and records all **explicit deferrals** to future versions. Its purpose is to:

- Prevent scope creep during documentation or execution
- Eliminate ambiguity around "small additions"
- Preserve privacy, audit integrity, and performance

No implementation is authorized through this document.

## 3. In-Scope Summary (v1.1-B)

The following capabilities are **explicitly included** in v1.1-B:

- VP-only Notes Module
- Linked notes (Guest / Appointment / Case)
- Stand-alone personal notes
- Soft delete for notes
- Full audit of note actions
- Read-only dashboard widgets for notes

No additional capabilities are implied.

---

## 4. Explicit Deferrals (Not in v1.1-B)

The following items are **explicitly excluded** from v1.1-B and deferred to future consideration:

### Notes Enhancements
- Rich formatting (blocks, tables, markdown)
- Attachments within notes
- Note templates
- Pinned, starred, or categorized notes

### Collaboration & Sharing
- Shared notes
- Secretary or team access
- Delegation or co-editing

### Search & Intelligence
- Global notes search
- Cross-entity search
- Analytics, summaries, or insights

### Input Methods
- Handwriting or pen input
- Voice notes
- OCR or handwriting-to-text

### Platform & Integration
- Offline notes
- PWA-specific behavior
- External integrations or exports

---

## 5. Rationale for Deferrals

Deferrals are intentional to:

- Protect executive privacy
- Avoid premature complexity
- Maintain performance and security
- Prevent notes from becoming a task or document system

---

## 6. Change Control Rules

- Any attempt to introduce a deferred item requires:
  - New epic definition
  - Updated scope document
  - Explicit approval

Verbal requests or UI-only additions are invalid.
